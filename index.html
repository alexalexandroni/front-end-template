Front-end Template

Why do we need a Front-end Template?

Starting a new project, most of us were in a position to ask ourselves: how should I start? what tools should I use? how should I structure the files (CSS, images, fonts, JavaScript)? and many other similar questions. This tool - Front-end Template - tries to answer these questions by containing some of the most popular and useful tools on the web that are applicable to all browsers.
It can help you start a new project much easier!

What does Front-end Template contains?

It uses HTML5 Boilerplate structure that already includes:

  • the new HTML5 elements
  • normalize.css for CSS normalizations across all browsers (Chrome, Firefox, IE8+, Opera, Safari)
  • jQuery (the most used JavaScript library) and Modernizr for feature detection
  • and other stuff that you can check out in the code provided here

In addition we included:

  • selectivizr which emulates CSS3 pseudo-classes and attribute selectors in IE6-8
  • cross browser HTML5 placeholder script located in main.js
  • and a responsive grid system based on the Twitter Bootstrap Grid System, but more easy to use.

Simple grid system

This is a modified version of Twitter Bootstrap Grid System.

The main difference between Twitter Bootstrap Grid System and this one contains in:

  • a practical use with less columns: 1, 2, or 3
  • use of descriptive classes like .column-one-third(33%), .column-half(50%), .column-third(75%) or .column-full(100%)
  • a simple, easy to modify and use grid system

Look to the examples below to see how the grid works and how you can apply it to your code.
